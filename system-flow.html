System logic map

How a user moves from signup to outcome.

This page shows the relationship cycle at the behavior level: how people enter discovery, like and vet each other, move into one active match, respond to deadlines, exit cleanly, or receive Shuns for poor follow-through.

Core rule

One live match locks discovery.

Likes remain stored, but browsing, new match requests, and new conversations are paused until the active match closes.

01

Signup and Entry

User creates an account, completes onboarding, and becomes eligible for discovery.

Incomplete onboarding: browse locked Complete onboarding: enter discovery
Eligible users move into the same intent pool
02

Discovery Pool

The user sees available people who pass the basic product filters.

  • Same intent pool only.
  • Unavailable if either person already has an active match.
  • Hidden or banned accounts are excluded.
  • Already-seen profiles are removed from that user's discovery list.
03A

Pass

User says no. The swipe is recorded, the profile leaves discovery, and no other action is required.

03B

Like With Interest Score

User likes someone and must attach a 1 to 10 interest score. This creates an incoming like for the other person.

Score is visible when the like is reviewed No active match is created yet
The liked person reviews the incoming like
04

Incoming Like Review

The recipient can ignore it, close it, or decide they want to move forward.

No interest

The like remains history. No match is made.

Interested back

The recipient sends a match request to confirm intent before locking both people.

A match request creates the final confirmation step
05

Match Request

The original liker receives a request. Confirming it creates the active match.

Request ignored

No active match. Both remain free to continue discovery unless another match is active.

Request confirmed

A match opens, both profiles are locked into one active connection, and other pending requests touching either user are closed.

Confirmed match starts a timed relationship track
06

Active Match Lock

Both users now focus on one person. They can chat, send intro video, plan a date, request unmatch, or report.

Browse locked Likes still visible but paused New match requests locked Conversation opens for this match
07

Intro Video

Each person must submit a short intro video.

Deadline: 24 hours from match creation
Both submit

Match advances to date planning.

Timer expires

Missing person receives a Shun. If both miss it, both receive a Shun and the match closes.

08

Plan First Date

Either person can propose a time and location. The other can confirm or counter. The proposed date must happen before the deadline.

Deadline: 14 days from match creation
Both accept a plan

Match advances to the final decision window.

Timer expires

Person who failed to participate receives a Shun. If neither participates, both receive a Shun and the match closes.

09

Final Decision

After date planning is complete, each person chooses the outcome.

Deadline: 28 days from match creation
Attract + Attract

Success outcome. Match closes cleanly and testimonial/story can be created.

Not a fit + Not a fit

Mutual close. No Shun assigned.

Mixed or Shun decision

Negative/non-mutual outcome. Match closes as Shun and both users receive a decision Shun in the current prototype.

Timer expires

Missing decision-maker receives a Shun. If both miss it, both receive a Shun and the match closes.

A

Unmatch Request

Either person can request to end the match during any active stage. The other person gets 24 hours to answer.

Other says yes, mutual

Match closes. No Shun.

Other says no, not mutual

Requester receives an unmatch-not-mutual Shun.

No response in 24 hours

Responder receives an unmatch-no-response Shun.

B

Report and AI Review

A user can report a recent match. The intended moderation path is AI-assisted review of chat history, report details, and match context.

Report dismissed

No user penalty. Report remains in history.

Reported user at fault

User receives a moderation Shun or, for severe harassment, account status can become banned.

Reporter is abusing reports

System should support assigning the penalty to the reporter after AI/moderation determines bad-faith reporting.

Accountability

Shun score categories to track.

The Shun score is the visible accountability count. Each Shun increases the user's total and is stored by reason.
